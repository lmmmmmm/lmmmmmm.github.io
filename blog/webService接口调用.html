# 客户端webService接口调用

# 一、名词

  1. WSDL：WSDL是Web Service的描述语言，用于描述WebService及其函数、参数和返还值。
  2. SOAP：简单对象访问协议（Simple Object Access Protocol），用于交换XML，Web Service通过HTTP协议发送请求和接收结果时，内容都是用XML格式封装的，并增加了一个特定的HTTP消息头用于说明HTTP消息的内容格式，这些特性的HTTP消息头和XML内容格式就是SOAP协议。SOAP协议=HTTP协议 + XML数据格式。

# 二、调用过程

  1. 客户端把调用的方法和参数转换成符合WSDL定义的XML文档（SOAP消息）
  2. 通过HTTP协议发送给服务器
  3. 服务器接收并解析XML，提取数据
  4. 服务器执行方法
  5. 服务器将执行方法的返还值转换成符合WSDL定义的XML
  6. 通过HTTP协议将XML发放给客户端
  7. 客户端接收并解析XML文档，并提取数据

# 三、 使用IDEA生成代码

1、 在IDEA中右击项目名称，选择webService->Generate Java Code From Wsdl。

2、在弹出的页面中，Web Service Wsdl Url中输入或者选择wsdl文件所在位置（文件可以是在本地，也可以是在服务器上），Package prefix输入文件所在包，Web Service Platform 选择Glassfish.... 那个选项。最后点击ok即可。

3、选择完成后，可以在我们之前选择的包下看到生成了很多Java类与Class文件，我们可以将class文件删除。

4、我们可以在一个main方法中对这些接口进行测试，new 一个生成的xxxWsService类，然后调用其getxxxWsServiceImport方法。最后在调用webService中的接口即可完成调用。

 public static void main(String[] args) {
     OrganQueryWsService queryWsService = new OrganQueryWsService();
     IOrganQueryWsService organQueryWsServiceImplPort = 			                    			queryWsService.getOrganQueryWsServiceImplPort();
     String res = organQueryWsServiceImplPort.getOrganByIds("3500", "123123");
     System.out.println(res);
}

# 四、 使用Apache cxf生成

在pom文件中引入cxf的依赖或者导入cxfjar包

<dependency>
    <groupId>org.apache.cxf</groupId>
    <artifactId>cxf-spring-boot-starter-jaxws</artifactId>
    <version>3.2.7</version>
</dependency>

public static void main(String[] args) {
    String url = "wsdl文件所在位置"；
    String methodName = "调用接口名称";
    String param = "参数";
    JaxWsDynamicClientFactory dcf = JaxWsDynamicClientFactory.newInstance();
    Client client = dcf.createClient(url);
    Object[] a = new Object[0];
    try {
        a = client.invoke(methodName, param);
    } catch (Exception e) {
        e.printStackTrace();
    }
    System.out.println(a[0]);
}
