# Scrapy常用命令

# 1 创建项目

scrapy startproject projectName

创建成功后进入项目,项目结构如下所示:

├── scrapy.cfg
└── testproject
    ├── __init__.py
    ├── items.py
    ├── middlewares.py
    ├── pipelines.py
    ├── __pycache__
    ├── settings.py
    └── spiders
        ├── __init__.py
        └── __pycache__

# 2 生成Spider

# 2.1 基本生成方式

scrapy genspider baidu www.baidu.com

该命令一共两个参数 scrapy genspider [args1] [agrs2]:第一个参数将生成爬虫的名称,第二个参数表示要爬取的网站.执行完成后,将在spiders文件夹下生成一个python文件,名称则为刚生成spider的第一个参数.该文件如下所示:

# -*- coding: utf-8 -*-
import scrapy


class BaiduSpider(scrapy.Spider):
    name = 'baidu'
    allowed_domains = ['www.baidu.com']
    start_urls = ['http://www.baidu.com/']

    def parse(self, response):
        pass

# 2.2 带模板的生成方式

在生成spider时scrapy为我们提供了模板:我们可以使用以下命令来查看:

scrapy genspider -l

我们可以看到它提供了以下模板:

  • basic
  • crawl
  • csvfeed
  • xmlfeed
    这样我们在生成spider时可以加上这些模板,如:
scrapy genspider -t crawl zhihu www.zhihu.com

然后我们就可以在spider文件夹下看到zhihu.py文件

# -*- coding: utf-8 -*-

# 3 运行sprider

scrapy crawl spidername

spidername 为spider的名称,如刚生成的baidu,如果会打印出一些请求百度的日志,则说明Spider运行成功.

# 4 检查代码是否有误

scrapy check

# 5 查看项目中所有spider

$ scrapy list
baidu
zhihu

# 6 fetch

帮助我们访问一个网页,将源代码返还.如:

scrapy fetch http://www.baidu.com

# 6.1支持参数

加上nolog参数将会直接返还源代码,不返还一些请求信息

scrapy fetch --nolog http://www.baidu.com

加上headers将会返还response Header

 fetch  --headers  http://www.baidu.com

# 7 shell

可以使我们以交互模式访问一个网站,如:

scrapy shell https://www.baidu.com

# 8 获取当前配置信息

获取settings.py中的配置信息:如

 scrapy settings --get MONGO_DB
