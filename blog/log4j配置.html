# Log4j配置

# 1 配置根Logger

log4j.rootLogger=level,appenderName1(自定义)，appenderName2(自定义)

# 1.1 level

​ level表示日志级别：可设的值为：OFF 、FATAL 、ERROR、 WARN 、INFO 、DEBUG 、ALL。Log4建议使用中间加粗的四个级别。级别顺序为：OFF > FATAL > ERROR＞WARN ＞INFO ＞DEBUG＞ALL。既假如配置level为INFO ，则DEBUG级别日志不显示，INFO 级别以上的日志如WARN等会显示。

# 1.2 appenderName

​ appenderName表示日志内容要输出到哪里，appenderName为自定义输入，可以输入console，file，A1，B1等任意内容，可以同时指定多个输出地并用逗号隔开。

# 1.3 根Loggeer配置例子

log4j.rootLogger=INFO,console,logFile

# 2 配置日志信息输出地

log4j.appender.appenderName=className

# 2.1 appenderName

​ appenderName为我们在根Logger中配置的appenderName

# 2.2 className

​ className为我们要选择日志的输出地，Log4j给我们提供以下常用的类作为配置日志输入地。

org.apache.log4j.ConsoleAppender				#将日志输出到控制台中
org.apache.log4j.FileAppender      				#将日志输出到文件中
org.apache.log4j.DailyRollingFileAppender       #每天产生一个日志文件
org.apache.log4j.RollingFileAppender       	    #文件达到指定尺寸产生一个新文件
org.apache.log4j.WriterAppender        	        #将日志信息已流格式发送到任意指定地方

# 2.3 日志输出地基本配置

  • ConsoleAppender配置

    Threshold=WARN 			 #指定日志信息的最低输出级别，默认为DEBUG
ImmediateFlush=true		 #所有消息都会被立即输出，设为false则不立即输出。默认为true
Traget=System.err		 #默认值为System.out

  • FileAppender配置

    File=D:/longing.log      #指定日志输出位置
Threshold=WARN           #指定日志信息的最低输出级别，默认为DEBUG
ImmediateFlush=true		 #所有消息都会被立即输出，设为false则不立即输出。默认为true
Append=false			 #true表示追加到指定的文件中，false表示覆盖原文件中的内容，默认为true

  • RollingFileAppender配置

    Threshold=WARN           #指定日志信息的最低输出级别，默认为DEBUG
ImmediateFlush=true		 #所有消息都会被立即输出，设为false则不立即输出。默认为true
Append=false			 #true表示追加到指定的文件中，false表示覆盖原文件中的内容，默认为true
File=D:/longing.log      #指定日志输出位置
MaxFileSize=10MB		 #在日志达到指定大小自动回滚，后缀可以为KB,MB,GB
MaxBackupIndex=2		 #指定可以产生滚动文件的最大数。

    例如设置 File=D:/longing.log，MaxFileSize=10MB，MaxBackupIndex=2，表示当文件日志文件超过10MB时，则会将生成longing.log.1、longing.log.2、和longing.log 文件

  • DailyRollingFileAppender配置

    Threshold=WARN           #指定日志信息的最低输出级别，默认为DEBUG
ImmediateFlush=true		 #所有消息都会被立即输出，设为false则不立即输出。默认为true
Append=false			 #true表示追加到指定的文件中，false表示覆盖原文件中的内容，默认为true
File=D:/longing.log      #指定日志输出位置
DatePattern='.'yyyy-MM   #每月产生一个新的日志文件

    例如设置DatePattern='.'yyyy-MM，则当月文件名为longing.log，前一个月文件名则为longing.log .yyyy-MM。也可以指定按天、周、日等来分别滚动日志文件。如：

    • '.'yyyy-MM 每月
    • '.'yyyy-ww 每周
    • '.'yyyy-MM-dd 每天
    • '.'yyyy-MM-dd-a 每天两次
    • '.'yyyy-MM-dd-HH 每小时
    • '.'yyyy-MM-dd-mm 每分钟

# 3 配置日志信息格式

log4j.appender.appenderName.layout=className

# 3.1 appenderName

​ appenderName为我们在根Logger中配置的appenderName

# 3.2 className

​ className为我们要选择日志的信息格式，Log4j给我们提供以下常用的类作为配置日志信息格式。

org.apache.log4j.HTMLLayout					     	#已HTML形式布局
org.apache.log4j.PatternLayout      				#可以灵活的指定布局模式
org.apache.log4j.SimpleLayout      		   		    #包含日志信息的级别和信息字符串
org.apache.log4j.TTCCLayout       	   			    #包含日志产生的时间、线程、类别等新

# 3.3 日志格式基本配置

  • HTMLLayout

    LocationInfo=true                #输出java文件名和行号，默认为false
Title=My Logging				 #默认值为Log4j Log Message
